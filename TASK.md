# Voiceline Backend Task

## Requirements
* Quickly set up Go as backend (preferably Gin) which receives audio data and sends it over to a LLM api of your choice.
* What the LLM then actually does and supplies back to go is of your choice.
* Submit the returned data directly to any API, like a note taking tool, a CRM or google sheets.
* Please also include a short README or other explanation, so its easy for us to quickly spin up.
* Try to follow best practices in code but do not overengineer it!

## Other guidelines
* Consider Cost efficeny and scalability
* Do not hard code and use configurable setup
* Add unit tests to cover all ciritial areas and differt paths
* Use const over hard code
* Use layered architecture with clean separation
* Use DDD when possible
* Naming should be high quality and readable
* Security should be considered
* Consider time and space complexity efficiency
* Add comments when needed.
* Pass context when go rotine is used. 
* Handle errors efficiently, avoid fatal.
